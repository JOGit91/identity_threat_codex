Device Code Phishing
Critical
Credential Access · Initial Access

Attackers abuse the OAuth 2.0 device authorization flow to trick users into handing over a valid Microsoft 365 token — no fake login page, no suspicious URL. The victim completes MFA on Microsoft's real servers and the attacker receives a persistent refresh token.

Device Code OAuth MFA Bypass Token Theft APT29 No Phishing Domain
MITRE T1528 · T1566 View profile →
OAuth Consent Abuse
Critical
Persistence · Credential Access

Attackers register malicious OAuth applications and trick users into consenting to broad M365 permissions. The app gains persistent, silent access to mailboxes and files that survives password resets, MFA changes, and session revocations.

OAuth Consent Phishing Persistence Mail Access BEC Graph API
MITRE T1550.001 · T1528 View profile →
MFA Fatigue / Push Bombing
High
Initial Access · Defense Evasion

Attackers with stolen credentials flood a victim with MFA push notifications until they approve one out of fatigue or confusion. No phishing page required — just a phone number and a password. Often paired with a vishing call impersonating IT support.

Push Bombing MFA Abuse Social Engineering Scattered Spider Lapsus$ Vishing

Mailbox Rule Manipulation
High
Persistence · Exfiltration · Defense Evasion

After gaining mailbox access, attackers create inbox rules to forward email to external addresses and silently delete security alerts — giving them persistent intelligence and hiding the compromise from the victim. Survives password resets if not explicitly remediated.

About This Resource

Built for defenders and M365 admins

Threat information targeting Microsoft 365 environments moves fast. New attack techniques, new threat actors, new tooling — it's difficult to build a coherent picture of what you're actually up against.

This codex documents each known attack technique in depth: what it is, how it works, what evidence it leaves behind, who uses it, and most importantly — what you can do about it. Each entry is designed to be actionable, not just educational.

Hunt queries are written for Microsoft Sentinel and Microsoft Defender XDR. Prevention guidance is prioritized so you know where to start.
